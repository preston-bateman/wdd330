Chapter 8 CSS3 Transforms and Transitions

Transforms

  • The CSS3 transform property lets you lets you translate, rotate, scale, and/or skew any element on the page.
  • Transform function manipulate elements. Transform property is one or more transform functions
  • Don't use multiple transform properties. Instead use mulitple functions seperated by a space.
  • Translate functions let you move things up, down, left, and right from current position. transform: translate(x,y)
  • Transforms don't work on inline elements. You can fix with inline-block
  • The scale function scales things horizontally then vertically. If only one statement is passed then it is used for both. Scaling also doesn't affect the flow of the document. Other elements won't move to fit the element
  • Rotate rotates an element around it's center a certain amount. Deg, grads, radians, or turns can be used.
  • Skew will skew an element on the x and y axis. If only one number is provided, it will only do the x axis.
  • Transform-Origin property changes the point from which the transform occurs. It uses the same syntax as background-position.

Transitions

  • Always declare the transform in the default state
  • You can add as many css properties to the transition property seperated by commas.
  • Transition duration tells how long it takes for the change. It can by in seconds or milliseconds. 200 ms is very typical.
  • Transition-timing-function lets you do non-linear time transitions. Use values like: ease, linear, ease-in, ease-out, ease-in-out. You can also use the value of a cubic-bezier function to alter the transition timing.
  • The transition delay function lets you delay the animation. Can use S or MS.
  • The shorthand for the 4 transition properties is: transition: propertyValue durationValue timingFunctionValue delayValue;

Animations

  • Animations are different from transitions because you have control over the midpoinst and not just the beginning and end.
  • You control the animation with keyframes. A keyframe is a snapshot or a point of transition.
  • You create an animation by defining it first with an @keyframes with the name of the property and curly braces and then you assign it to an element
  • Syntax: @keyframes animationName {keyframes-selector {css-styles;}}
  • Keyframe selectors are either from or to, or a percentage of where it is in the animation.
  • You attach an animation to an element using the animation-name property. Don't put quotes around the name in the value.
  • animation-duration defines how long the animation takes from 0 to 100 percent.
  • animation-timing-function is like the transition one but for animations.
  • animation-iteration-count lets you define how many times the animation will play through. Default is 1 but you can do as many as infinite.
  • animation-direction lets you determine if it goes start to finish and restart or plays back to first and then first to back. Use normal, reverse, alternate, or alternate-reverse. When animations are played in reverse the timing functions are also reversed. ease-in becomes ease-out.
  • Animation-play-state property defines whether an animation is running or paused. This allows you to control the animation with css or javascript.
  • The animation property takes as its value a space-separated list of values for the longhand animation-name, animation-duration, animation-timing-function, animation-delay, animation-iteration-count, animation-direction, animation-fill-mode, and animation-play-state properties

Chapter 12 Canvas, SVG, and Drag and Drop

Canvas

  • HTML5 has a Canvas API that lets you drow anything using javascript.
  • With canvas, we can draw shapes and lines, arcs and text, gradients and patterns. In addition, canvas gives us the power to manipulate pixels in images and even video.
  • You have to add a canvas element to your html document. Text inbetween canvas tags will only show if canvas is not supported by the browser.
  • You need to set the width and height in the canvas tag and not using css as this defines the canvas coordinate system size. You also need to be able to grab the element using the DOM in JavasScript. That means id and class attributes.
  • Tip: giving canvas a border in css allows you to see the edges of the canvas object.
  • Grab the canvas element by assigning it to a variable in JavasScript.
  • Setup the canvas context with let context = canvasObj.getContext("2d"). This is an instance of the context object.
  • 3d drawing is available with the WebGL API. Documentation: WebGL
  • context has a strokeStyle property and a fillStyle property that are set by giving them a string color, CanvasGradient object or a CanvasPattern object. You can use any CSS color name, hexdec value or rgba value.
  • CanvasPattern
  • there is a fillRect and strokeRect method that take the arguments x, y, width and height. Coordinates in canvas start in the top left corner.
  • There is also a drawing method with a path that begins with a context.beginPath() and finishes with a context.closePath(). Between the calling of the 2 methods you can call other methods like .arc.

SVG

  • SVG is a vector graphic that maintains its quality when scaled
